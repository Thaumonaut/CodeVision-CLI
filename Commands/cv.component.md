# /cv.component

Goal: Define, document, or revise a reusable component pattern (Spec §7.2).

## Rules
- **Reuse-first**: Check `components/registry.md` before defining anything new.
- If defining a new component, it must include:
  - Purpose
  - Inputs / Props / API
  - Outputs / Events
  - Accessibility constraints (if UI)
  - Styling tokens used
- **Output**: Update `components/registry.md`. Provide FILE blocks for the component implementation if the user is in the engineering phase.

Include the Style + Question + Artifact-first + Write protocol from `_STYLE.md`.
